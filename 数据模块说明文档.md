# `crypto_quant/data` 数据模块说明文档

本文档说明 `crypto_quant/data` 数据模块的定位、文件结构、时间处理规则、CSV 加载、OKX 行情拉取、数据清洗校验、`DataFeed` 增强方法，以及如何统一从 CSV / OKX / MySQL 获取回测数据。

---

## 1. 数据模块定位

数据模块位置：

```text
crypto_quant/data/
```

它负责把不同来源的数据统一转换成框架内部使用的：

```text
BarData -> DataFeed -> 策略 / 回测引擎
```

当前支持的数据来源：

```text
1. 本地 CSV；
2. OKX / ccxt；
3. MySQL 数据库 repository。
```

crypto_quant/data/

| `source.py` | 统一 CSV / OKX / MySQL 到 `DataFeed` 的入口 |
| `__init__.py` | 对外导出常用类和函数 |

## 3. 时间处理规则

OKX接口返回的 K 线时间是时间戳，通常是毫秒时间戳，例如：

```text
1717200000000
```

而平时人工查看、CSV 文件、策略研究中更常用的是北京时间 `datetime`，例如：

```text
2024-06-01 08:00:00
```

因此数据模块现在统一提供时间工具：

```python
from crypto_quant.data import (
    BEIJING_TIMEZONE,
    UTC_TIMEZONE,
    datetime_to_milliseconds,
    timestamp_to_datetime,
    to_milliseconds,
)
```

### 3.1 默认约定

当前默认约定是：

```text
1. OKX 时间戳按 UTC 时间戳解析；
2. 无时区 datetime 默认按北京时间理解；
3. fetcher 默认返回北京时间 datetime；
4. 如果需要保留 UTC，可以手动传 target_timezone=UTC_TIMEZONE；
5. 如果需要无 tzinfo 的 datetime，可以传 drop_timezone=True。
```

### 3.2 时间戳转北京时间

```python
from crypto_quant.data import BEIJING_TIMEZONE, timestamp_to_datetime

value = timestamp_to_datetime(1717200000000, target_timezone=BEIJING_TIMEZONE)
```

### 3.3 北京时间转毫秒时间戳

```python
from datetime import datetime
from crypto_quant.data import datetime_to_milliseconds

since = datetime_to_milliseconds(datetime(2024, 6, 1, 8, 0))
```

如果传入的是无时区 `datetime`，默认按北京时间处理。

## 4. `BarData` / `DataFeed` 核心结构

`BarData` 表示一根 K 线：

```python
BarData(
    symbol="BTC/USDT",
    timeframe="1m",
    datetime=datetime(...),
    open=Decimal("100000"),
    high=Decimal("101000"),
    low=Decimal("99000"),
    close=Decimal("100500"),
    volume=Decimal("12.5"),
)
```

`DataFeed` 是多根 K 线组成的数据源，会拆出：

```python
data.open
data.high
data.low
data.close
data.volume
data.datetime
```

策略里可以写：

```python
self.data.close[0]
self.data.close[-1]
self.data.close[-2]
```

含义分别是：

| 写法 | 含义 |
|---|---|
| `self.data.close[0]` | 当前 K 线收盘价 |
| `self.data.close[-1]` | 上一根 K 线收盘价 |
| `self.data.close[-2]` | 上两根 K 线收盘价 |

## 5. `DataFeed` 新增实用方法

### 5.1 当前游标

```python
data.cursor
```

表示当前推进到第几根 K 线。

### 5.2 当前可用 K 线数量

```python
data.available_bars
```

策略中可以用：

```python
if self.data.available_bars < 20:
    return
```

### 5.3 重置游标

```python
data.reset()
```

把 `DataFeed` 和所有 `DataLine` 的 cursor 重新设为 `-1`。

### 5.4 获取历史窗口

```python
bars = data.window(20)
closes = data.close.window(20)
```

表示取当前 K 线向前数 20 根。

### 5.5 切片

```python
sub_data = data.slice(100, 200)
```

返回新的 `DataFeed`。

### 5.6 转成 DataFrame

```python
df = data.to_dataframe()
```

字段包括：

```text
symbol, timeframe, datetime, open, high, low, close, volume
```

## 6. CSV 加载

使用：

```python
from crypto_quant.data import load_bars_from_csv

数据 = load_bars_from_csv(
    path="data/BTC_USDT_1m.csv",
    symbol="BTC/USDT",
    timeframe="1m",
)
```

默认需要 CSV 至少包含：

```text
datetime, open, high, low, close
```

`volume` 可选，没有会自动填 `0`。

### 6.1 CSV 时间处理

如果 CSV 中是：

```text
2024-06-01 08:00:00
```

且没有时区信息，模块默认把它理解为北京时间。

如果 CSV 中是时间戳：

```text
1717200000000
```

模块会自动识别毫秒时间戳。

### 6.2 自定义时区

```python
from crypto_quant.data import UTC_TIMEZONE, load_bars_from_csv

数据 = load_bars_from_csv(
    path="data/BTC_USDT_1m.csv",
    symbol="BTC/USDT",
    timeframe="1m",
    target_timezone=UTC_TIMEZONE,
)
```

## 7. 数据清洗和校验

数据清洗入口：

```python
from crypto_quant.data import clean_and_validate_bars

cleaned_bars, report = clean_and_validate_bars(bars)
```

当前会检查：

```text
1. 时间统一到目标时区；
2. 重复 K 线去重；
3. 按 symbol / timeframe / datetime 排序；
4. open/high/low/close 是否大于 0；
5. volume 是否小于 0；
6. high 是否小于 low；
7. open/close 是否落在 high/low 范围内；
8. 是否存在缺失 K 线间隔。
```

返回的 `DataQualityReport` 包含：

| 字段 | 中文说明 |
|---|---|
| `input_count` | 输入 K 线数量 |
| `output_count` | 清洗后 K 线数量 |
| `duplicate_count` | 去掉的重复 K 线数量 |
| `missing_intervals` | 疑似缺失 K 线的时间区间 |

如果希望缺失 K 线时报错，可以传：

```python
clean_and_validate_bars(bars, raise_on_missing=True)
```

## 8. OKX 批量拉取历史 K 线

`MarketDataFetcher` 现在支持按起止时间自动分页拉取：

```python
from datetime import datetime
from crypto_quant.data import MarketDataFetcher
from crypto_quant.enums import KlineInterval

fetcher = MarketDataFetcher(client)

bars = fetcher.fetch_ohlcv_range(
    symbol="BTC/USDT",
    timeframe=KlineInterval.M1,
    start=datetime(2024, 6, 1, 8, 0),
    end=datetime(2024, 6, 2, 8, 0),
    limit=1000,
)
```

这里的 `start` 和 `end` 如果是无时区 `datetime`，默认按北京时间理解。

也可以直接返回 `DataFeed`：

```python
data = fetcher.fetch_data_feed(
    symbol="BTC/USDT",
    timeframe="1m",
    start=datetime(2024, 6, 1, 8, 0),
    end=datetime(2024, 6, 2, 8, 0),
)
```

### 8.1 请求间隔

如果想降低请求频率：

```python
bars = fetcher.fetch_ohlcv_range(
    symbol="BTC/USDT",
    timeframe="1m",
    start=datetime(2024, 6, 1, 8, 0),
    end=datetime(2024, 6, 2, 8, 0),
    request_interval_seconds=0.2,
)
```

## 10. `MarketDataPipeline` 数据闭环管道

现在数据模块新增了：

```python
from crypto_quant.data import MarketDataPipeline
```

它用于把 OKX 历史 K 线准备流程串起来：

```text
OKX 批量拉取
        ↓
清洗和校验
        ↓
写入 MySQL
        ↓
从 MySQL 读取 DataFeed
```

示例：

```python
from datetime import datetime
from crypto_quant.data import MarketDataPipeline

pipeline = MarketDataPipeline(fetcher, market_repository)
result = pipeline.fetch_clean_store(
    symbol="BTC/USDT",
    timeframe="1m",
    start=datetime(2024, 6, 1, 8, 0),
    end=datetime(2024, 6, 1, 9, 0),
)
```

返回的 `result` 包含：

| 字段 | 中文说明 |
|---|---|
| `symbol` | 交易对 |
| `timeframe` | K 线周期 |
| `fetched_count` | 从 OKX 拉取到的 K 线数量 |
| `stored_count` | 清洗后写入数据库的 K 线数量 |
| `data_feed` | 从数据库读回来的 `DataFeed` |
| `quality_report` | 数据质量报告 |

注意：这个管道正式目标是 MySQL，因为 `MarketDataRepository.upsert_klines()` 使用 MySQL 的 `ON DUPLICATE KEY UPDATE`。

## 11. 推荐数据流程

更完整的数据闭环可以是：

```text
OKX 批量拉取
        ↓
清洗和校验
        ↓
写入 MySQL
        ↓
从 MySQL 读取 DataFeed
        ↓
回测引擎
        ↓
保存回测结果
```

对于本地研究，也可以是：

```text
CSV
 ↓
清洗和校验
 ↓
DataFeed
 ↓
回测引擎
```

## 12. 当前边界

当前数据模块已经能支撑更完整的数据准备流程，但仍有边界：

```text
1. MarketDataPipeline 已能串联 OKX 拉取、清洗、入库和读取，但仍需要用户提供 MySQL 配置；
2. 多 symbol / 多 timeframe 的统一对齐还没有实现；
3. 缺失 K 线目前只报告区间，不自动补齐；
4. fetcher 批量拉取依赖交易所限制，实际大批量下载时仍建议分阶段保存；
5. 实盘 websocket 行情还没有接入。
```
