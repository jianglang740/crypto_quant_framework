# Strategy Research

这个目录用于研究加密货币量化策略。

这里的代码可以是探索性的，但研究结果最终应该尽量回到框架中，通过现有的数据、回测、数据库和仪表盘链路验证。

## 目录划分

```text
research/
├── scripts/      # 一次性研究脚本、数据观察、参数实验
└── candidates/   # 有潜力但尚未整理成正式框架策略的策略原型
```

## 使用边界

- `research/scripts/`：适合快速验证想法，例如波动统计、指标观察、参数扫描。
- `research/candidates/`：适合放已经比较像策略的原型代码。
- 成熟策略不长期停留在这里，后续应整理为框架正式策略，并通过回测、入库、仪表盘观察和测试网验证。

## 研究原则

- 优先复用框架已有模块，不单独造一套数据和回测系统。
- 研究脚本可以不追求工程化，但关键假设、参数和结论要写清楚。
- 不在研究脚本里放 API key、数据库密码、私钥等敏感信息。
- 不把研究区变成正式实盘入口；实盘或测试网运行仍放在 `real/`。

## 推荐流程

```text
提出策略想法
→ 在 scripts/ 做数据观察和统计验证
→ 在 candidates/ 写策略原型
→ 用框架回测并写入数据库
→ 在 dashboard/ 观察结果
→ 稳定后再进入 real/ 做测试网验证
```
