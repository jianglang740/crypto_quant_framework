# 数据库验证过程总结

这份文档记录本次对 `crypto_quant_framework` 数据库部分的验证结果。

验证目标不是学习 SQL 语法本身，而是确认：

```text
Python 程序
↓
SQLAlchemy / PyMySQL
↓
MySQL
↓
框架的 repository 方法
↓
真实业务数据写入和读回
```

这条链路是否真的跑通。

---

## 1. 本次验证的总体结论

本次已经确认：**框架与本地 MySQL 的核心数据库交互是通的**。

已经验证通过的闭环包括：

```text
程序连接 MySQL
程序自动建表
程序写入 strategy_runs
程序从 strategy_runs 读回记录
程序写入 klines
程序从 klines 读回 DataFeed
MySQL 行情数据喂给 BacktestEngine 回测
回测结果保存回 MySQL
VS Code MySQL 插件可视化确认数据真实存在
```

这说明当前框架的数据库部分不是“只写了表结构但没有接入业务”，而是已经能够进入真实的数据流和回测流。

## 3. 本次验证说明了什么

### 3.1 程序和数据库不是脱节的

本次验证证明：

```text
框架里的 repository 方法
不是摆设
不是伪代码
不是只定义了表结构
```

它们已经可以真实操作 MySQL。

例如：

```python
repo.create_run(...)
```

本质上对应：

```sql
INSERT INTO strategy_runs (...)
VALUES (...);
```

而：

```python
repo.list_runs(...)
```

本质上对应：

```sql
SELECT ... FROM strategy_runs ...;
```

只是框架用 Python 方法和 ORM 对象封装了 SQL 操作。

---

### 3.2 repository.py 是程序和数据库之间的“翻译层”

可以这样理解：

```text
程序世界：BarData / DataFeed / BacktestResult / StrategyRun
数据库世界：klines / strategy_runs / orders / trades / equity_curve
```

`repository.py` 的作用就是在两边之间翻译。

例如：

```text
BarData -> klines 表
klines 表 -> DataFeed
BacktestResult -> orders/trades/equity_curve 表
```

所以学习这个项目的数据库部分时，重点不是先背 SQL，而是理解：

```text
哪个 repository 方法
负责把哪个 Python 对象
写入哪张 MySQL 表
```

---

### 3.3 MySQL 已经可以作为回测数据源

本次最重要的验证之一是：

```text
MySQL klines 表 -> DataFeed -> BacktestEngine
```

这说明以后可以形成标准流程：

```text
Binance 拉 K 线
↓
清洗校验
↓
写入 MySQL klines
↓
从 MySQL 读出 DataFeed
↓
跑回测
```

目前虽然用的是假 K 线，但数据库链路本身已经验证成功。

后面只需要把“假 K 线”替换成“真实 Binance K 线”。

---

### 3.4 回测结果可以被长期保存和复盘

以前回测结果如果只在终端打印，程序结束后就不方便追踪。

现在通过：

```python

```

可以把一次回测保存成数据库记录，包括：

```text

```

这意味着以后可以做：

```text

```

这也是后期仪表盘的基础。

## 6. 对当前学习阶段的意义

本次验证解决的是数据库学习中最关键的问题：

```text
程序到底怎么和数据库交互？
```

通过这次验证，可以把理解从“数据库像 Excel 表一样存数据”推进到：

```text
Python 对象
↓
repository 方法
↓
SQLAlchemy Session
↓
ORM 模型
↓
MySQL 表
```

也就是说，数据库不再只是手动打开插件增删查改，而是已经成为程序工作流的一部分。

当前最重要的理解是：

```text
models.py 定义表长什么样
session.py 负责连接、提交、回滚、关闭
repository.py 负责把业务对象写入/读出数据库
recorder.py 负责实盘/模拟实盘过程中的持续记录
```

本次已经重点验证了：

```text
models.py + session.py + repository.py
```

下一阶段再验证：

```text
MarketDataPipeline
LiveDatabaseRecorder
```
