# 项目依赖库说明

这份文档用于帮助你理解 `crypto_quant_framework` 当前使用到的依赖库。理解这些库的作用，是读懂整个框架的基础。

当前项目依赖可以分为两类：

1. 第三方依赖：需要通过 `pip install -r requirements.txt` 安装；
2. Python 标准库：Python 自带，不需要额外安装。

## 2. ccxt

### 2.1 ccxt 是什么？

`ccxt` 是一个统一封装全球多个交易所 API 的 Python 库。

它可以让你用比较统一的方式访问不同交易所，例如：

- Binance
- OKX
- Bybit
- Coinbase
- Kraken

本项目当前只封装 Binance。

### 2.2 为什么需要 ccxt？

如果不用 `ccxt`，你需要自己处理：

- Binance REST API 地址；
- 请求签名；
- API key 认证；
- 时间戳；
- 请求参数；
- 现货和合约接口差异；
- 返回数据格式解析。

使用 `ccxt` 后，可以直接调用：

```python
exchange.fetch_ohlcv(...)
exchange.fetch_balance(...)
exchange.create_order(...)
exchange.cancel_order(...)
```

这样项目可以把重点放在策略、回测和交易逻辑上。

### 2.3 本项目在哪里用了 ccxt？

主要在：

```text
crypto_quant/exchange/binance_client.py
crypto_quant/data/fetcher.py
```

在 `binance_client.py` 中：

```python
import ccxt

self.exchange = ccxt.binance(config.ccxt_options())
```

这行代码创建了 Binance 交易所客户端。

### 2.4 BinanceConfig 如何影响 ccxt？

位置：

```text
crypto_quant/config.py
```

核心代码：

```python
def ccxt_options(self) -> dict[str, Any]:
    default_type = "spot" if self.trading_mode == TradingMode.SPOT else "future"
    options = {"defaultType": default_type, **self.options}
```

含义：

| 项目配置 | ccxt 行为 |
|---|---|
| `TradingMode.SPOT` | `defaultType="spot"`，使用 Binance 现货接口 |
| `TradingMode.FUTURE` | `defaultType="future"`，使用 Binance USD-M 合约接口 |

### 2.5 ccxt 在数据获取中的作用

位置：

```text
crypto_quant/data/fetcher.py
```

示例：

```python
rows = self.client.exchange.fetch_ohlcv(
    symbol,
    timeframe_value,
    since,
    limit,
    params or {},
)
```

`ccxt.fetch_ohlcv()` 返回原始 K 线数组，通常类似：

```python
[
    [timestamp, open, high, low, close, volume],
    ...
]
```

框架会把它转换成自己的 `BarData`：

```python
BarData(
    symbol=symbol,
    timeframe=timeframe_value,
    datetime=...,
    open=Decimal(...),
    high=Decimal(...),
    low=Decimal(...),
    close=Decimal(...),
    volume=Decimal(...),
)
```

这样后面的回测引擎和策略不需要直接依赖 ccxt 的原始格式。

### 2.6 ccxt 在交易中的作用

位置：

```text
crypto_quant/exchange/binance_client.py
```

常用封装：

```python
fetch_balance()
fetch_positions()
create_order()
cancel_order()
fetch_order()
fetch_open_orders()
close_position()
set_leverage()
set_margin_mode()
```

例如下单最终会走：

```python
self.exchange.create_order(
    symbol,
    order_type.value,
    side.value,
    amount,
    price,
    request_params,
)
```

也就是说：

```text
策略调用 self.buy()
→ StrategyBase 生成 OrderRequest
→ LiveEngine 调用 BinanceClient.create_order()
→ BinanceClient 调用 ccxt.create_order()
→ ccxt 请求 Binance
```

### 2.7 读源码时你需要重点理解的 ccxt 概念

| 概念 | 含义 |
|---|---|
| `symbol` | 交易对，例如 `BTC/USDT` |
| `timeframe` | K线周期，例如 `1m`、`1h`、`1d` |
| `fetch_ohlcv` | 获取 K 线 |
| `fetch_ticker` | 获取 ticker |
| `fetch_order_book` | 获取订单簿 |
| `fetch_balance` | 获取账户余额 |
| `create_order` | 创建订单 |
| `cancel_order` | 撤销订单 |
| `params` | 交易所特殊参数，例如 `reduceOnly`、`positionSide` |

## 3. SQLAlchemy

### 3.1 SQLAlchemy 是什么？

`SQLAlchemy` 是 Python 生态里非常常用的数据库工具。它最重要的能力之一是 ORM。

ORM 的意思是：

```text
Object Relational Mapping
对象关系映射
```

你可以简单理解为：

```text
Python 类  ↔  MySQL 表
Python 对象 ↔  MySQL 表中的一行记录
```

### 3.2 为什么需要 SQLAlchemy？

如果不用 SQLAlchemy，保存 K 线可能需要手写 SQL：

```sql
INSERT INTO klines (...) VALUES (...)
```

而使用 SQLAlchemy，可以用 Python 类描述表结构：

```python
class Kline(Base):
    __tablename__ = "klines"
```

然后通过 Python 对象或语句操作数据库。

### 3.3 本项目在哪里用了 SQLAlchemy？

主要在：

```text
crypto_quant/database/models.py
crypto_quant/database/session.py
crypto_quant/database/repository.py
```

### 3.4 models.py 的作用

位置：

```text
crypto_quant/database/models.py
```

这个文件定义数据库表。

当前有四张表：

| Python 类 | MySQL 表名 | 用途 |
|---|---|---|
| `Kline` | `klines` | 保存 K 线 |
| `OrderRecord` | `orders` | 保存订单 |
| `TradeRecord` | `trades` | 保存成交 |
| `EquityCurve` | `equity_curve` | 保存权益曲线 |

例如：

```python
class Kline(Base):
    __tablename__ = "klines"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    symbol: Mapped[str] = mapped_column(String(32))
    timeframe: Mapped[str] = mapped_column(String(16))
    open_time: Mapped[datetime] = mapped_column(DateTime)
    open: Mapped[Decimal] = mapped_column(Numeric(36, 18))
```

这段代码的意思是：

```text
创建一张 klines 表，里面有 id、symbol、timeframe、open_time、open 等字段。
```

### 3.5 常见 SQLAlchemy 类型

本项目用了这些 SQLAlchemy 类型：

| 类型 | 对应数据库含义 | 项目用途 |
|---|---|---|
| `String(32)` | 字符串 | symbol、side、status |
| `DateTime` | 时间 | K线时间、成交时间 |
| `Numeric(36, 18)` | 高精度数字 | 价格、数量、成交额、权益 |
| `Text` | 长文本 | 保存订单原始 JSON |
| `Index` | 索引 | 防止重复 K 线 |

### 3.6 为什么价格字段用 Numeric？

因为数据库里也要避免浮点误差。

项目里 Python 计算使用：

```python
Decimal
```

数据库里对应使用：

```python
Numeric(36, 18)
```

这可以保证价格、数量、手续费、盈亏这类字段尽量精确。

### 3.7 session.py 的作用

位置：

```text
crypto_quant/database/session.py
```

主要负责创建数据库连接：

```python
def create_mysql_engine(config: MySQLConfig) -> Engine:
    return create_engine(config.url, echo=config.echo, pool_pre_ping=True, future=True)
```

以及创建 Session 工厂：

```python
def create_session_factory(engine: Engine) -> Callable[[], Session]:
    return sessionmaker(bind=engine, autoflush=False, autocommit=False, future=True)
```

你可以理解为：

```text
Engine = 数据库连接引擎
Session = 一次数据库操作上下文
```

### 3.8 repository.py 的作用

位置：

```text
crypto_quant/database/repository.py
```

Repository 是对数据库读写的进一步封装。

例如：

```python
repo = MarketDataRepository(session)
repo.upsert_klines(bars)
```

它的好处是：

```text
业务代码不用关心具体 SQL 怎么写，只调用 repo 方法即可。
```

### 3.9 读源码时你需要重点理解的 SQLAlchemy 概念

| 概念 | 含义 |
|---|---|
| `DeclarativeBase` | ORM 模型基类 |
| `Mapped` | SQLAlchemy 2.0 的类型标注写法 |
| `mapped_column` | 定义表字段 |
| `create_engine` | 创建数据库连接引擎 |
| `sessionmaker` | 创建 Session 工厂 |
| `Session` | 数据库操作会话 |
| `commit` | 提交事务 |
| `refresh` | 从数据库刷新对象 |
| `insert(...).on_duplicate_key_update` | MySQL upsert 写法 |

## 4. PyMySQL

### 4.1 PyMySQL 是什么？

`PyMySQL` 是 Python 连接 MySQL 的底层驱动。

SQLAlchemy 本身是 ORM/数据库工具，但它需要一个具体驱动去连接数据库。

本项目的数据库 URL 是：

```python
mysql+pymysql://username:password@host:port/database?charset=utf8mb4
```

这里的：

```text
+pymysql
```

表示 SQLAlchemy 使用 PyMySQL 作为 MySQL 驱动。

### 4.2 你会直接写 PyMySQL 代码吗？

当前项目不会。

项目中不会出现类似：

```python
import pymysql
```

因为我们是通过 SQLAlchemy 间接使用它。

你只需要知道：

```text
PyMySQL 是 SQLAlchemy 连接 MySQL 所需要的底层驱动。
```

### 4.3 如果没有安装 PyMySQL 会怎样？

当你创建 MySQL engine 时，可能会报错：

```text
ModuleNotFoundError: No module named 'pymysql'
```

解决：

```bash
python3 -m pip install PyMySQL
```

或者直接安装全部依赖：

```bash
python3 -m pip install -r requirements.txt
```

## 6. dataclasses

### 6.1 dataclasses 是什么？

`dataclasses` 是 Python 标准库，用于快速定义“主要存数据”的类。

不用 dataclass 时，你可能要写：

```python
class Account:
    def __init__(self, cash, equity, available):
        self.cash = cash
        self.equity = equity
        self.available = available
```

用了 dataclass 后：

```python
@dataclass(slots=True)
class Account:
    cash: Decimal
    equity: Decimal
    available: Decimal
```

Python 会自动帮你生成 `__init__`、`__repr__` 等方法。

### 6.2 本项目为什么大量使用 dataclass？

因为量化框架里有很多“数据对象”：

- `BarData`
- `Account`
- `Position`
- `OrderRequest`
- `Trade`
- `BacktestConfig`
- `PerformanceReport`

这些类主要是为了清晰表达数据结构，用 dataclass 很合适。

### 6.3 slots=True 是什么？

例如：

```python
@dataclass(slots=True)
class Account:
    ...
```

`slots=True` 可以：

- 减少对象内存占用；
- 防止随便给对象添加不存在的字段；
- 让数据结构更稳定。

## 7. decimal.Decimal

### 7.1 Decimal 是什么？

`Decimal` 是 Python 标准库 `decimal` 中的高精度十进制数字类型。

量化交易中，不建议用普通 `float` 表示价格、金额和数量。

例如：

```python
0.1 + 0.2
```

可能得到：

```python
0.30000000000000004
```

而：

```python
Decimal("0.1") + Decimal("0.2")
```

结果是：

```python
Decimal("0.3")
```

### 7.2 本项目为什么用 Decimal？

项目中这些字段都适合用 Decimal：

```text
open / high / low / close
volume
amount
price
fee
cash
equity
pnl
commission_rate
slippage_rate
```

因为这些都和交易资金或数量有关。

### 7.3 使用 Decimal 的注意事项

推荐这样写：

```python
Decimal("0.01")
Decimal("10000")
```

不推荐：

```python
Decimal(0.01)
```

因为 `0.01` 先被 Python 当成 float，可能已经产生精度误差。

## 10. typing

### 10.1 typing 是什么？

`typing` 用于类型标注。

例如：

```python
def fetch_balance(self, params: dict[str, Any] | None = None) -> dict[str, Any]:
```

它告诉读代码的人：

```text
params 可以是 dict，也可以是 None
返回值是 dict
```

### 10.2 类型标注有什么好处？

- 更容易读懂函数输入输出；
- IDE 可以提供更好的提示；
- 后续可以使用 mypy/pyright 做静态检查；
- 减少误用函数的概率。

### 10.3 Any 是什么意思？

`Any` 表示“任意类型”。

项目里和 ccxt 返回值相关的地方经常用 `Any`，因为交易所 API 返回结构复杂，不适合一开始就写得特别死。

作用是：

```text
每轮实盘循环结束后暂停几秒，再获取下一批数据。
```

## 16. pandas / numpy / matplotlib 在当前框架里的定位

当前框架仍然是交易框架骨架，重点是：

```text
交易所 API 封装
数据对象
策略生命周期
回测引擎
实盘引擎
数据库层
绩效统计
```

现在已经引入了 `pandas` 和 `numpy`，但用途比较克制：

| 库 | 当前用途 | 后续可能扩展 |
|---|---|---|
| `pandas` | 读取 CSV、转换 datetime、转换数值列 | rolling 指标、resample、多周期处理、研究分析 |
| `numpy` | 检查 CSV 数值是否为有效数字 | 向量化计算、波动率、相关矩阵、参数优化 |
| `matplotlib` | 当前还未引入 | 权益曲线、回撤曲线、交易点位图 |

也就是说，当前 `pandas` / `numpy` 主要服务于：

```text
本地 CSV 数据 → DataFeed → 回测引擎
```

暂时还没有把整个策略和回测框架改成 notebook 风格或向量化研究框架。

后续如果要增强研究分析能力，可以继续新增：

```text
crypto_quant/indicators/
crypto_quant/data/converter.py
crypto_quant/analysis/plotting.py
```

## 18. 一句话总结

当前项目依赖很克制：

```text
ccxt 负责交易所
SQLAlchemy 负责数据库抽象
PyMySQL 负责连接 MySQL
pandas 负责读取和清洗本地 CSV
numpy 负责数值有效性检查
Decimal / dataclass / enum / datetime / uuid 等标准库负责稳定表达交易系统里的核心对象
```

先理解这些库，就能更顺利地读懂当前框架。当前 `pandas` / `numpy` 已经用于 CSV 数据加载，后续还可以继续引入 `matplotlib` 或扩展指标模块来增强研究分析、指标计算和可视化能力。
