# orderbook_research 依赖库说明

## 运行依赖

```bash
pip install requests websocket-client
```

---

## requests

**用途**：拉取订单簿 REST 快照

```python
import requests
resp = requests.get(SNAPSHOT_URL, proxies={"https": PROXY}, timeout=15)
```

- 发送 HTTP GET 请求到 `fapi.binance.com/fapi/v1/depth`
- 通过 `proxies` 参数支持 SOCKS5 代理，传 `{"http": ..., "https": ...}` 即可
- 代理支持经过多年生产验证，在 SSH 隧道场景下稳定

---

## websocket-client

**用途**：订阅币安 WebSocket 增量深度流

```python
import websocket
ws_app = websocket.WebSocketApp(WS_URL, on_message=..., on_error=..., on_open=...)
ws_app.run_forever(proxy_type="socks5", http_proxy_host="127.0.0.1", http_proxy_port=1080)
```

- 订阅 `fstream.binance.com/ws/btcusdt@depth`，实时接收订单簿差量更新
- 运行在独立线程（`threading.Thread`），不阻塞主线程
- 代理通过 `run_forever` 的参数传入，不需要额外依赖

## 标准库（无需安装）

| 模块 | 用途 |
|---|---|
| `json` | 解析 WebSocket 推送的 JSON 消息 |
| `queue.Queue` | 线程安全地将 WS 消息从后台线程传给主线程 |
| `threading` | 在独立线程中运行 WebSocket 长连接 |
| `time` | 等待连接建立、记录时间戳 |
| `urllib.parse` | 解析 PROXY URL，提取 host 和 port |
| `dataclasses` | 定义 `LiquidityMetrics` 数据类 |

## 曾经尝试但放弃的库

| 库 | 放弃原因 |
|---|---|
| `aiohttp` | 在 SSH SOCKS5 隧道场景下 ProxyConnector 不稳定，反复超时 |
| `aiohttp-socks` | 不支持 `socks5h://` scheme，且与当前 aiohttp 版本有兼容问题 |
| `websockets`（asyncio） | `proxy` 参数在新版本行为不一致，WS 握手超时 |
| `python-socks` | 手动建 socket 传给 websockets 可以解决 WS，但 REST 侧问题未解决 |

## 代理使用说明

Mac 本地通过 SSH 隧道访问币安时：

```bash
# 启动隧道
ssh -fN quant-proxy

# 验证连通性
curl --socks5-hostname 127.0.0.1:1080 https://fapi.binance.com/fapi/v1/time
```

然后在脚本中设置：

```python
PROXY = "socks5://127.0.0.1:1080"
```

服务器上直连时：

```python
PROXY = ""
```
